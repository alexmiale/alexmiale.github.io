  • xG and EPV Modelling - Spring 2026

    Using Python and R, Komdean Masoumi and I created two models that predict the likelihood of a goal being scored in a game of soccer from any given space on the field. EPV, or expected possession value is the likelihood of a goal being scored in that team's possession, based on the positions and speed of every player, the position of the ball, and the phase of play. xG, or expected goals, is the likelihood a goal is going to be scored given a shot is being taken. In our model, we included other variables such as the angle to the goal, the number of players pressuring the ball, etc. Both models were trained on data from SkillCorner, specifically 10 matches of the Australian A-League, but used different aspects of the data, so two different preprocessing pipelines were created in Python. Then, the EPV model was created in Python using PyTorch, and the xG model was created in R using a manually implemented Monte-Carlo Markov Chain.

  • Resume Prediction - Spring 2026

    This project conducts experiments with modelling resume to job comparisons using different embeddings and natural language processing (NLP) techniques. We looked to match a resume with a target job and rate the comparison a continuous scale of 0 to 1. We began by establishing a baseline using TF-IDF embeddings and a Linear Regression model, before moving to relativized DOLMA embeddings and experimenting with a Deep Averaging Network, a LSTM, a hybrid BiLSTM, and finishing with a transformer from MiniLM. All preprocessing and model experimentation was done in Python using PyTorch and other standard data science libraries like numpy, pandas, and scikit-learn.

  • Song Recommendation System - Spring 2025

    For the final project in the Practical Neural Networks, my group created a system that would recommend similar songs upon providing an input song. The goal for this project was to create recommendations that weren't based on subjective metrics or popularity. The data used for this project was the DEAM dataset, a collection of 1802 songs used initially for emotional analysis in the mid 2010s. This data was already preprocessed using openSMILE, which provided us with information about valence, arousal, fundamental and partial frequencies, etc. For our network, we created an autoencoder and a variational autoencoder. We started with a normal autoencoder to extract the embeddings layer and carry out cosine similarity between each of the songs. We added a variational autoencoder to obtain a continuous latent space, but still extracted the embeddings layer and performed cosine similarity. The final models were deployed in a website on GitHub Pages that is accessible in the repository.

  • Recidivism Bias - Fall 2024

    This was my final project for my Artificial Intelligence class where I analyzed the bias in the popular COMPAS dataset. This project was inspired by a ProPublica article that identified the bias, and I decided to apply Aequitas to attempt to mitigate it. Aequitas is an open-sourced toolkit that is used for auditing models and making models more fair across sensitive attributes. In this case, I used the DataRepairer class that balances the spread of data such that, when comparing across a given attribute, the shapes of the predicted value graphs were the same. I was able to use Aequitas successfully and mitigate the models' bias, while showing how the bias-variance tradeoff works in machine learning. I also wrote a paper for this project, which is available in the 'Writings & Essays' page or in the repository.
